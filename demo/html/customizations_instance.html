Instance Customizations

Per grid instance customization is possible at the header, body and the footer level. Below are a few examples.

Custom header and body cells

When you start adding additional cells in either the header or the body, there is one rule to remember:
The grid will match the position of the header template cells with the ones in the body through the fields identified by the field-name attribute. Everything that is not bound to a field will be matched by position as well, and the extra non-field elements will turn into columns placed at the far right end of the grid.

Tapping into the currently bound row item can be done through the

  • gridItem scope variable, which points to the actual data item, and
  • gridDisplayItem, which points to the formatted item and should only be used for presentation purposes.

More examples can be found on this site in the Column Definition section.

                    Customizing data cells
                

                

                    

                        You can override the style of the generated data cells, but do remember to match by field-name the TD element with a TH element present in the header. 

                        The data cell template must be placed inside an HTML child element, otherwise the content will be ignored and treated as a placeholder for the default cell template. 

                        Keep in mind that you have access to both the gridItem scoped variable, which points to the actual data item, and gridDisplayItem, which points to the formatted item and should only be used for presentation purposes.

                    Angular directives on TR elements
                

                

                    

                        You can apply a limited amount of directives onto the TR elements.
                        In general, DOM changing directives such as ng-if, ng-repeat or ng-switch are not supported.
